BEM - Block Element Modifier

Una metodología para nombrar clases CSS de forma consistente y escalable

¿Qué es BEM?

BEM es una metodología que divide el CSS en tres conceptos: Bloques (componentes independientes), Elementos (partes del bloque) y Modificadores (variaciones).

Ventajas
  • Código altamente legible
  • Fácil reutilización de componentes
  • Escalable a proyectos grandes
  • Fácil mantenimiento
Nomenclatura
.button { }
.button__icon { }
.button--primary { }
Ejemplo de código HTML con BEM 
<!-- Componente Button con BEM -->
<button class="button button--primary">
  <!-- Elemento icono dentro del botón -->
  <i class="button__icon"></i>
  Guardar Cambios
</button>

OOCSS - Object Oriented CSS

Separación de estructura y presentación para CSS orientado a objetos

Principios OOCSS
1. Separar Estructura y Presentación

Crea clases reutilizables que describan solo la estructura, separadas de las clases que describen el estilo visual.

2. Separar Contenedor e Hijo

Los estilos no deben depender de la ubicación. Un componente debe verse igual independientemente de dónde esté.

Ejemplo OOCSS 
/* Estructura reutilizable */
.container { }

/* Presentación reutilizable */
.bg-primary { }
.text-white { }
.rounded { }

/* Combinar en HTML */
<div class="container bg-primary 
      text-white rounded"></div>

SMACSS - Scalable and Modular Architecture

Arquitectura escalable y modular basada en categorías de CSS

Base

Estilos por defecto para elementos HTML (body, h1, p, a, etc.) Incluye reset.css y normalización.

Layout

Estilos para la estructura general del sitio (header, footer, grid, etc.) Componentes macro del layout.

Module

Componentes reutilizables (botones, cards, modales, etc.) Los elementos más comunes y modulares.

State

Estilos para estados de componentes (activo, deshabilitado, hover, etc.) Cambios de estado con .is-*

Theme

Estilos relacionados con temas (colores, tipografía) Permite cambiar apariencia completa del sitio.

Responsive

Media queries y breakpoints para diferentes dispositivos Adaptación a móvil, tablet y desktop.

│   ├── _buttons.scss
│   ├── _cards.scss
│   └── _forms.scss
├── state/
│   └── _state.scss
├── theme/

└── main.scss

Buenas Prácticas en CSS

Recomendaciones para escribir CSS profesional y mantenible

1 Usar nombres significativos

Los nombres de clases deben ser descriptivos y fáciles de entender. Evita nombres genéricos como "box" o "container".

/* Bien */
.card-header { }

/* Mal */
.h { }
2 Evitar selectores específicos

Mantén la especificidad baja para evitar conflictos. Prefiere clases sobre IDs y selectores complejos.

/* Bien */
.button { }

/* Mal */
#header .nav ul li a { }
3 Organizar en componentes

Divide tu CSS en módulos pequeños y reutilizables. Cada componente debe ser independiente y autosuficiente.

/* Separar por componentes */
buttons.css, cards.css, forms.css
4 Usar preprocesadores

SASS/SCSS permite variables, mixins y funciones que hacen el CSS más mantenible. Compila a CSS limpio.

// Variables SCSS
$primary: #0d6efd;

¿Por qué aprender estas metodologías?

Las metodologías CSS modernas como BEM, OOCSS y SMACSS son esenciales para el desarrollo profesional:

  • Escalabilidad: Permite que tus estilos crezcan sin convertirse en un caos
  • Colaboración: Otros desarrolladores entienden tu código fácilmente
  • Mantenimiento: Cambios futuros son más fáciles y seguros
  • Performance: CSS organizado es más eficiente y se carga más rápido
  • Profesionalismo: Demuestra que trabajas como un desarrollador senior
